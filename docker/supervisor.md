`docker run --restart=always ...`

# 3.容器内使用suprvisor启动服务

supervisor启动服务，如果进程down了会自动抬起。

>详细见[supervisor官网](http://supervisord.org/)

supervisor的相关命令

```
CMD ["supervisord","-n","-c","/etc/supervisord.conf"]    #Dockerfile里配置的启动命令，-n -c为前台运行模式
supervisorctl status        #查看所有进程的状态
supervisorctl stop demo       #停止demo,conf里配置的名称
supervisorctl start demo      #启动demo
supervisorctl restart demo     #重启demo
supervisorctl update        #配置文件修改后使用该命令加载新的配置
supervisorctl reload        #重新启动配置中的所有程序
```

Dockerfile里安装supervisor
```
RUN yum install -y epel-release && yum install -y python-pip \
    && pip install supervisor==3.1.3 && pip install meld3==1.0.1 \
    && mkdir -p /etc/supervisor/conf.d \
    && yum remove -y epel-release && yum remove -y python-pip
COPY ["supervisor/supervisord.conf","/etc/supervisord.conf"]
COPY ["supervisor/demo.conf","/etc/supervisor/conf.d/demo.conf"]

```

demo.conf
```
[program:demo]         #名称
autorestart=True       
autostart=True
redirect_stderr=True
stdout_logfile_maxbytes=20MB
stdout_logfile_backups=20
stdout_logfile=/release/demo/log/demo.log
redirect_stderr=true
directory=/release/demo   #运行命令的目录
command=java -jar demo.jar    #运行命令，注意此命令需为前提运行模式，不要使用nohup后台启动
```
